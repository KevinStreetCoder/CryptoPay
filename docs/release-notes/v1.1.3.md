## Long-form (for cpay.co.ke/changelog)

### What's new in v1.1.3

**Stable in-place upgrades** — Up to and including v1.1.2, every locally-built APK was signed with a fresh ephemeral debug keystore. Android refuses to install a different-signed APK over an existing install ("App not installed as package appears to be invalid"). v1.1.3 introduces a permanent release keystore (RSA-4096, 10000-day validity, SHA-1 `73:21:C5:C0:91:4D:9B:75:18:AF:31:E2:19:E9:8D:1E:EE:4E:84:49`) so every future build upgrades v1.1.3 cleanly. Users who previously installed v1.1.2 from the website must uninstall once, install v1.1.3, then all subsequent versions upgrade in place.

**Massive payout-rail expansion via SasaPay channels** — The B2C path now resolves any of 40 channels: M-Pesa (`63902`), Airtel Money (`63903`), T-Kash (`63907`), SasaPay Wallet (`0`), plus 36 Kenyan banks (KCB, Equity, Co-op, ABSA, NCBA, DTB, Stanbic, Standard Chartered, I&M, CBA, Family, NBK, Prime, Sidian, Spire, Consolidated, Credit Bank, Development Bank, Ecobank, GT Bank, Gulf, Habib, HFC, Housing Finance, Jamii Bora, M Oriental, Mayfair, Middle East, NIC, Paramount, SBM, UBA, Victoria Commercial, Bank of Africa, Bank of Baroda, Bank of India). Internally accessible via `apps.mpesa.sasapay_client.SASAPAY_CHANNELS`; user-visible UI for picking the destination rail ships in v1.1.4.

**Hosted card / Airtel checkout** — New `checkout_payment()` client method posts to SasaPay's `/payments/card-payments/` endpoint and returns a hosted CheckoutUrl that supports Visa/Mastercard, M-Pesa STK, Airtel Money, and SasaPay Wallet in one form. Backend-only in this release; mobile UI follows.

**Defense-in-depth callback verification** — Every SasaPay payment callback is now (a) HMAC-SHA512 verified against the field-concat message keyed on Client ID per the canonical docs, (b) source-IP-checked against the 10 documented `/32` SasaPay hosts plus 127.0.0.0/8, and (c) re-queried against SasaPay's Transaction Status API before any wallet credit. The status-API check is gated on `SASAPAY_VERIFY_CALLBACKS_VIA_API=True` (default on). A spoofed callback that somehow passed signature + IP would still fail because SasaPay's own server has no record of the transaction.

**Auto-topup of Utility account before each B2C** — Before every B2C payout, the client probes Utility balance and moves Working → Utility (+5% buffer) if low. Eliminates the "insufficient funds in Utility" failure mode that plagued the first round of production payouts.

**Comprehensive SasaPay error code table** — Full classification of success / pending / failed codes per the docs. Pending codes (`PENDING`, `SP01001`, `SP01004`) no longer trigger compensation — they correctly stay open until status-API resolution. M-Pesa C2B + B2C codes, Pesalink, Airtel, and HTTP-level `SP.4xx`/`SP.5xx` codes all mapped to user-visible messages.

**Onboarding tour scroll fix** — The last step of the in-app tour was off-screen on small devices. The tour now scrolls the highlighted element into view before placing the tooltip.

**OTP resend live countdown** — Previously the "Resend code" button was always tappable, leading to rate-limit errors. Now the button shows a live `(57s)` countdown and is disabled until ready.

**Under the hood**
- Phase 1 KMS uplift · 5/7 production secrets migrated to Google Secret Manager (rotation-ready)
- Phase 2 KMS uplift · TOTP HOTP secrets are now KMS-wrapped via envelope encryption
- Cloudflare R2 object storage live for KYC docs + receipts (bucket `cpay-prod`, smoke tested)
- `SASAPAY_ALLOWED_IPS` default now lists the 10 documented `/32` hosts (was permissive)
- Build pipeline: stable keystore + versioned output filename (`cryptopay-v1.1.3-vc7.apk`) + EXPO_TOKEN no longer baked into the script

### Operational notes for ops

- **First install of v1.1.3 over v1.1.2** · users get "App not installed" because of the keystore change. They uninstall once. Every release after v1.1.3 upgrades cleanly.
- **The release keystore lives at `mobile/credentials/cpay-release.keystore`.** It is gitignored. Loss of this file = every existing install must uninstall to take a new APK. Two backups exist (password manager + offline encrypted USB).
- **`SASAPAY_VERIFY_CALLBACKS_VIA_API`** can be flipped to `False` if SasaPay's status API has an outage and we want to fall back to signature-only verification. Do not leave off in steady state.
- **`SASAPAY_ALLOWED_IPS`** must be widened if SasaPay rotates their callback origin IPs. Their docs list 10 hosts as of 2026-05-09.
